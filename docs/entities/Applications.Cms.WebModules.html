    Applications.Cms.WebModules Entity

    Namespace: Applications.Cms

    The abstract basic building block of web sites. Each module can render multiple web pages. Entity: Cms_Web_Modules

    Default Visualization

    Default Display Text Format:
    {Name:T}
    Default Search Member:
    Name

    Aggregate

    An aggregate is a cluster of domain objects that can be treated as a single unit.

    Aggregate Parent:

    Attributes

    Name Type Description
    Id guid
    IsPublished boolean Specifies whether the module is published and will be showed in the web site. Required Default(false) Filter(eq)
    LocalUrl string (128) nullable The local Module Url. The full Url of a module is formed based on the sites Base Url, the local Urls of all parent modules and the module Local Url. Null means that the web Module is the default web module within the parent path. Filter(like)
    ModulePicture byte[] nullable Default picture. Used for picture links, small module icons, etc. PNG format is suggested.
    ModuleType ModuleType Specifies the content handler. The content handler is responsible for generating the actual web page content. Module Types include Category, Static, News, Product, etc. Required Filter(like)
    Name string (512) Multilanguage Module name. This is used as a title when displaying the web Module. Required Filter(like)

    References

    Parent WebModules (nullable) The parent web Module. It can only be a module with Module Type = Category. Null means that the web Module is root web Module. Filter(multi eq)
    WebSite WebSites The web site to which the module belongs. Required Filter(multi eq) Owner
