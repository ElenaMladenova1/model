    Table Crm_Offers

    Offers (quotes) for sales of products. Can contain variants and choice groups. The user can select some approved lines and generate Sales Order. Entity: Crm_Offers

    Owner Tables Hierarchy

    • Gen_Documents

    Summary

    Name Type Description
    Offer_Id uniqueidentifier PK
    Document_Id uniqueidentifier
    Deal_Id uniqueidentifier The opportunity (deal) on which this offer is based.
    Store_Id uniqueidentifier The store, from which the products will be sold. When NULL, the store is unknown or N/A.
    Customer_Id uniqueidentifier The customer to which the products are offered.
    Price_List_Id uniqueidentifier The price list, based on which the prices are selected. Initially copied from the definition of the Customer.
    Required_Delivery_Date date When not NULL, specifies required delivery date for all offer lines. When the lines contain different delivery dates, this is NULL.
    Notes nvarchar(254)
    Document_Currency_Id uniqueidentifier The currency in which the document is issued. All amounts are in this currency
    Payment_Type_Id uniqueidentifier Way of payment. NULL means that no specific way of payment was offered
    Payment_Due_Date datetime The last term for payment
    Valid_Till datetime Last date when the offer is valid. NULL means the offer has no specific validity date
    Sales_Person_Id uniqueidentifier Responsible sales person. NULL when no sales person was assigned.
    Accepted_On datetime The date on which the customer accepted the offer. NULL if the offer is not yet accepted or the date is unknown.
    Recipient_Person_Id uniqueidentifier The offer recipient. It should be employee of the customer. NULL means that there is no default recipient or he/she is unknown
    Is_Single_Execution bit Readonly Specifies whether the document is a single execution of its order document.
    Is_Released bit Readonly True if the document is not void and its state is released or greater
    Ship_To_Customer_Id uniqueidentifier The customer to whom to ship the sales order. Usually it is a customer entry for a sub-party of the primary customer
    Row_Version timestamp
